        Функциональность:


        Server side:

a) Хоткей_1: выскакивает окно с запросом слушающего порта.

b) сервер должен слушать заданный TCP порт

b.a) все передаваемые сообщения должны кодироваться 
в виде escape последовательностей

c) при подключении клиента выслать ему исходное содержимое документа
содеримое документа упаковано в тег full:
Например:
<full>Тестовый документ 1,2,3,4,5</full>

d) При внесении изменения в документ, сервер должен разослать
всем клиентам информацию об изменениях
при добавлении: <add>1,2,3,4,text</add>
1 - номер строки с которой начинаются изменения
2 - номер столбца с которого начинаются изменения
3 - номер строки, на которой заканчиваются изменения
4 - номер столбца на котором заканчиваются изменения
text - сам текст

при удалении: <del>1,2,3,4</del>
1 - номер строки с которой начинаются удаление
2 - номер столбца с которого начинаются удаление
3 - номер строки, на которой заканчиваются удаление
4 - номер столбца на котором заканчиваются удаление

e) При получении от клиента запроса full или add/change, сервер
должен внести изменение в свой документ и разослать информацию
клиентам

f) Сервер периодически (раз в N секунд) должен рассылать всем клиентам
md5 полной версии документа для лучшей синхронизации
<md5>ae698hkd28nskad92034r023</md5>

g) При получении запроса <getfull></getfull> сервер должен высылать
полную версию документа




        Client side:

a) Хоткей_2: выскакивает окно с запросом адреса сервера и порта.

b) клиент должен подключиться к заданному серверу. Если не удалось,
уведомление об ошибке подключения.

с) При внесении изменения в документ, клиент должен отправить на сервер
либо целиком документ (full), либо информацию об изменении (add/del)

d) При получении от сервера запроса full или add/del, клиент должен
внести изменение в документ.

e) При получении от сервера md5 запроса, клиент должен проверить 
правильность документа, и, в случе, если докмент не верный,
отправить серверу запрос getfull
